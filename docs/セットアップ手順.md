## 1. Notion にコンテンツを入れる

### 占い師を追加
「Fortune Lab 占い師DB」に占い師を追加し、以下を入力：
- **名前**（必須）／ **id**（URL用の半角英数字。例: `mizuki`）
- 顔写真・肩書き・キャッチコピー・紹介文・占術・鑑定スタイル・料金目安・表示順
- **公開** にチェック → サイトに掲載されます

> 紹介文はプロパティに加えて、**ページ本文**にも自由に書けます（プロフィール下に表示）。

### ブログ記事を追加
「Fortune Lab ブログ記事DB」に記事を追加：
- **タイトル**・**著者**（占い師DBから選択）・**公開日**
- slug（URL。空ならNotionページIDから自動）・抜粋・カバー画像・タグ
- 本文は**ページ本文**に記述
- **公開状態 = 公開** → サイトに掲載されます

## 2. GitHub リポジトリを作成して公開

```bash
cd "fortune lab-site"
git init
git add .
git commit -m "Fortune Lab 初期構築"
git branch -M main
git remote add origin https://github.com/mayotesoh/fortune-lab-site.git
git push -u origin main
```

その後、GitHub 上で：
1. **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `NOTION_TOKEN` / Secret: `ntn_...`（`.env` と同じ値）
2. **Settings → Pages → Build and deployment → Source: GitHub Actions**
3. Actions が走り、`https://mayotesoh.github.io/fortune-lab-site/` で公開

> ⚠️ `.env` と `HANDOFF.md` は `.gitignore` 済み（トークンは公開されません）。

---

## 3. 予約フォームを有効化（GAS）

`gas/README.md` の手順に従って：
1. 予約用スプレッドシートを作成 → ID を `gas/Code.gs` に設定
2. Apps Script に `Code.gs` / `NotionSync.gs` を貼り付け
3. スクリプトプロパティに `NOTION_TOKEN` を設定
4. ウェブアプリとしてデプロイ → `.../exec` URL を取得
5. `src/consts.ts` の `GAS_URL` に貼り付けて push

---

## 4. （任意）LINE 連携

`src/consts.ts` で以下を設定すると LINE 関連機能が有効になります：
- `LINE_URL` … 公式LINEの友だち追加URL（フローティングボタン等が表示）
- `LIFF_ID` … LIFF予約（LINE内で名前自動入力）

対話式のLINE予約ボットは、Branch の `gas/LineBot.gs` を参考に追加できます。

## 反映タイミングの注意

静的サイトのため、**Notion の変更は再ビルドで反映**されます：
- 自動：`main` への push 時
- 定期：日本時間 朝9時・夜21時（`.github/workflows/deploy.yml` の cron）
- 手動：GitHub の Actions タブ →「Run workflow」
