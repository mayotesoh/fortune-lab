# 決済（Stripe 前払い）設定手順

予約フォームの「予約して支払いへ進む」で Stripe の決済ページに進み、
支払い完了後に予約が「支払済み」で確定します。

```
予約入力 → Stripe決済ページ → 完了ページ（入金確認）→ 予約確定（支払済み）
```

- 料金は **占い師DBの「鑑定料金」**（設定済み：碧井¥10,000／夜船¥8,000／沙織¥9,000）。
  おまかせ（占い師未指定）は既定額 ¥5,000（`Payment.gs` の `DEFAULT_PRICE`）。
- **`STRIPE_SECRET_KEY` を設定するまでは決済は動かず、従来どおり前払いなしで予約**を受け付けます（安全に段階移行できます）。

---

## 1. Stripe アカウント & キー

1. Stripe に登録（https://stripe.com/jp）
2. 画面右上が **「テスト環境」** になっていることを確認
3. 開発者 → APIキー → **シークレットキー**（`sk_test_...`）をコピー
   - シークレットキーは**サイトには絶対に置かない**（GASのみに保管）

## 2. GAS を更新して再デプロイ

Apps Script プロジェクト（予約フォームの送信先）を更新します。

1. **新規ファイル追加**：`Payment.gs` を作成し、[`gas/Payment.gs`](../gas/Payment.gs) を貼り付け
2. **差し替え**：[`gas/Code.gs`](../gas/Code.gs)、[`gas/NotionSync.gs`](../gas/NotionSync.gs) を最新の内容で上書き
3. **プロジェクトの設定 → スクリプト プロパティ** に追加：
   - キー：`STRIPE_SECRET_KEY`　値：`sk_test_...`
   - （既存の `NOTION_TOKEN` はそのまま）
4. **デプロイ → デプロイを管理 → 編集 → バージョン「新バージョン」→ デプロイ**
   ※ URL（`/exec`）は変わりません。

## 3. テスト（テストモードで通す）

1. サイトの予約フォームで占い師・日付・時間・メニューを選び「予約して支払いへ進む」
2. Stripe決済ページで**テストカード**を入力：
   - カード番号：`4242 4242 4242 4242`／有効期限：未来の日付（例 12/34）／CVC：任意3桁
3. 支払い後、完了ページに「ご予約が完了しました」が表示されればOK
4. 確認：
   - Notion「鑑定予約DB」に **決済状態=支払済み／金額／決済ID** が入った予約が作成される
   - Stripe ダッシュボード（テスト）の「支払い」に記録される

## 4. 本番へ切り替え

テストで問題なければ、Stripe を**本番環境**に切り替え、本番のシークレットキー
（`sk_live_...`）を `STRIPE_SECRET_KEY` に上書きして再デプロイするだけ。
**アカウントを後から変える場合もこのキーの差し替えだけ**で移行できます。

---

## 料金の変更

- 占い師ごと：Notion「占い師DB」の **鑑定料金** を編集（サイト再ビルド不要）
- おまかせ既定額：`Payment.gs` の `DEFAULT_PRICE`
- ※現在は「占い師ごとに1つの料金（全額前払い）」。メニュー別・時間別に分けたい場合は拡張が必要（要相談）。

## 補足・既知の制約

- **返金・キャンセル**：Stripe ダッシュボードから返金。予約DBのステータスを「キャンセル」にすると枠は自動で空きに戻る。
- **二重予約**：決済ページに進む直前に空き枠を再確認。まれに支払い直前で埋まった場合でも支払いは無駄にせず、備考に「⚠️枠重複の可能性」を付けて記録（手動調整）。
- **ブラウザを閉じた場合**：完了ページに戻らないと予約作成が走りません（支払いは Stripe に記録済み）。100%自動化するには **Stripe Webhook**（`checkout.session.completed` → GAS）の追加が有効。必要になったら実装します。
