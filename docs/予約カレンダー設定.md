# 予約カレンダー（占い師別の空き枠）設定手順

予約フォームで「占い師 × 日付」を選ぶと、その占い師の空いている時間だけが
表示され、二重予約も防止されます。仕組みは次の通りです。

```
空き枠 = 営業枠（曜日・時間） − 既存予約 − 前後のインターバル − 過去時刻（当日）
```

## 時間の指定と鑑定時間

- **開始時刻は10分刻み**（例：14:00 / 14:10 / 14:20 …）
- お客様は**鑑定時間（30分 / 1時間 / 1時間30分）を選択**します
- 選んだ時間ぶんの枠を確保します（1時間なら1時間ぶん埋まる）
- 連続鑑定にならないよう、**前後に10分のインターバル**を自動で空けます

例）13:00〜21:00 受付・既存予約が15:00から60分の場合、60分鑑定で選べるのは
`13:00〜13:50` と `16:10〜20:00`（14:00〜16:00 は確保・インターバルで選べない）。

設定を変えたい場合は `gas/Availability.gs` の先頭：
- `SLOT_MINUTES`（開始時刻の刻み。既定10分）
- `BUFFER_MINUTES`（インターバル。既定10分）
- `DURATION_OPTIONS`（選べる鑑定時間。既定 30/60/90分）※ `src/consts.ts` の `DURATIONS` と揃えること

コード側（サイト・GAS）は実装済みです。**下の2つの設定だけ**行えば有効になります。
設定前でも「既定営業時間（毎日 10:00〜21:30）」で動くので、予約は止まりません。

## 1. Notion「占い師DB」にプロパティを追加

占い師DB（`507fd75b0aa94c48a259d05b6b211ea4`）に、以下4つを追加します。
**プロパティ名は完全一致**させてください（型も合わせる）。

| プロパティ名 | 型 | 内容・例 |
|---|---|---|
| `受付曜日` | マルチセレクト | 受け付ける曜日。選択肢を 月/火/水/木/金/土/日 で作成。例：月・火・水・金 |
| `受付開始` | 数値 | 開始時刻の「時」。例：`10`（10:00開始） |
| `受付終了` | 数値 | 終了時刻の「時」。例：`22`（最終枠は 21:30） |
| `休業日` | テキスト | 単発の休み。`YYYY-MM-DD` をカンマ/改行区切りで複数可。例：`2026-08-13, 2026-08-14` |

- **未設定のままでOK**：空欄の項目は既定値（毎日 10:00〜22:00）で補完されます。占い師ごとに変えたい所だけ入れてください。
- 予約DBの「所要分」に鑑定時間が自動記録されます（空き枠の計算に使用）。
- 予約の集計には既存の「希望占い師（リレーション）」「日付」「時間」「ステータス」を使います。ステータスが `キャンセル` の予約は空き扱いに戻ります。

## 2. GAS を更新して再デプロイ

Apps Script プロジェクト（予約フォームの送信先）を更新します。

1. Apps Script エディタを開く
2. **新規ファイルを追加**：`Availability.gs` を作成し、リポジトリの
   [`gas/Availability.gs`](../gas/Availability.gs) の内容を貼り付け
3. 既存の **`Code.gs` を差し替え**：[`gas/Code.gs`](../gas/Code.gs) の内容で上書き
   （`SPREADSHEET_ID` はご自身の値を維持してください）
4. `NOTION_TOKEN`（スクリプトプロパティ）と、占い師DB がそのインテグレーションに
   コネクト済みであることを確認（予約DBと同じ設定でOK）
5. **デプロイ → デプロイを管理 → 編集（鉛筆）→ バージョン「新バージョン」→ デプロイ**
   ※ 新バージョンでデプロイしないと反映されません。URL（`/exec`）は変わりません。

### 動作確認
- エディタで `testAvailability` を実行 → ログに空き枠が出ればOK
- ブラウザで `〈GASのURL〉?action=slots&teller=&date=2026-07-25&duration=60` を開く →
  `{"status":"success","slots":[...]}` が返ればOK

## 補足・仕様

- **おまかせ（占い師=指定なし）**：占い師が未確定のため、既定営業枠を表示し、
  二重予約チェックは行いません（担当確定後に調整）。
- **二重予約防止**：送信時にもGAS側でロックを取りつつ再チェックします。同時アクセスで
  同じ枠が埋まった場合は「別の時間をお選びください」を表示。
- **APIに繋がらない場合**：フォームは従来の全枠（10:00〜21:30）を表示して予約自体は継続できます。
- 将来の拡張：LINE内予約（LIFF）でも同じ空き枠APIを利用可能。
